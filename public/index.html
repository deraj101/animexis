Animexis API

Anime streaming backend powered by Node.js, Express, MongoDB, and Redis. Scrapes Anitaku and enriches metadata via AniList GraphQL. JWT-secured, with subscriptions, comments, notifications, and background workers.

01 Authentication

Most write operations and user-specific data require a JWT token. Obtain one by registering or verifying OTP, then include it as a Bearer token.

STEP 01
Send Email
POST to /auth/send-otp with your email. A 6-digit code is sent.
STEP 02
Verify OTP
POST to /auth/verify-otp with email + code to receive a JWT.
STEP 03
Use Token
Attach header: Authorization: Bearer <token>
STEP 04
Admin Access
Admin JWT required for /admin/* routes. Email must be in ADMIN_EMAILS env.
ℹ️ Tokens carry the user's email as payload. All authenticated endpoints attach the full user object to req.user.
02 Rate Limits & Usage Limits
Global API Limit
500 req
Per IP, per 15-minute window. Applies to all /api/* routes.
Free Users — Episodes
20 / day
Unique episodes via /episode-info, /stream, /video-info. Tracked in Redis Sets per email per day.
Premium Users
Unlimited
No episode limit. Requires subscription: "premium" in user record.
Rate-Limit Exempt
/health, /ad
Health checks and the ad video endpoint are always excluded from rate limiting.
⚠️ Exceeding the daily episode limit returns HTTP 403 with limitReached: true. Premium upgrade via /api/payments/create-checkout-session.

08 Anime — Episode & Streaming
GET /anime/episode-info 🔐 JWT Get video sources — counts toward free limit
ParameterTypeRequiredDescription
url string required Episode URL from details endpoint (e.g. /anime-name-episode-1)
GET /anime/stream 🔐 JWT Proxy video stream — counts toward free limit
GET /anime/video-info 🔐 JWT Video metadata (content-type, size) — limited
GET /anime/:animeId/episode/:episodeNum Direct episode links — 5 min cache
GET /anime/ad Ad video stream (MP4, range-supported, no auth)
ℹ️ /stream proxies the video through your server for CORS compatibility. Supports HLS (.m3u8) and MP4 with byte-range requests.

13 Comments
GET /comments/:animeId Fetch all comments for anime/episode
GET /comments/count/:animeId Total comment count
POST /comments/create 🔐 JWT Post comment or reply (triggers REPLY notification)
POST /comments/:commentId/like 🔐 JWT Toggle like (triggers LIKE notification)
DELETE /comments/:commentId 🔐 JWT Delete comment + all direct replies
ℹ️ Pass ?episodeNum=3 to scope comments to a specific episode. Omit for anime-level discussion. The parentId field enables threaded replies.
14 Notifications
GET /notifications 🔐 JWT Fetch last 50 notifications + unread count
PATCH /notifications/:id/read 🔐 JWT Mark single notification as read
POST /notifications/read-all 🔐 JWT Mark all notifications as read

Notification types: REPLY · LIKE · RELEASE. All auto-pruned after 30 days.

15 Admin Routes
⚠️ All /admin/* routes require a JWT token belonging to an email listed in the ADMIN_EMAILS environment variable. Returns 403 otherwise.
GET /admin/stats 🛡 admin Total users, new today/week, active, banned — 1 min cache
GET /admin/recent-users 🛡 admin Latest registered users (up to 200)
GET /admin/activity 🛡 admin Recent activity log (ban, register, etc.)
GET /admin/top-anime 🛡 admin Top 10 most viewed anime
GET /admin/active-users 🛡 admin Users seen in last 5 minutes (live)
POST /admin/ban-user 🛡 admin Ban a user by email
POST /admin/unban-user 🛡 admin Unban a user by email
POST /admin/set-subscription 🛡 admin Manually set user to free or premium
POST /admin/set-otp-bypass 🛡 admin Enable or disable OTP bypass for a user
POST /admin/clear-cache 🛡 admin Flush all node-cache entries
GET /admin/monthly-visits 🛡 admin Monthly visit chart data
16 Payments — Stripe
POST /payments/create-checkout-session 🔐 JWT Create Stripe Checkout session (returns redirect URL)
Request Body
{ "priceId": "price_1Abc123..." }
GET /payments/sync-session/:sessionId 🔐 JWT Verify payment + instantly upgrade user (bypasses webhook delay)
POST /webhooks/stripe Stripe webhook — listens for checkout.session.completed
ℹ️ The webhook endpoint requires the raw request body for Stripe signature verification. It is mounted before the JSON body parser in app.js.
